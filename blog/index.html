AtomicReference

2023年5月23日 · 閱讀時間約 2 分鐘
Vincent

lambda語法中使用到local variable時，必須是effective final。

例如我們想在以下的List中，取得所有字數的總合，並且取得字數最長的字，以下是使用迴圈處理的範例。

List<String> names = Arrays.asList("Vincent", "David", "Ken", "Randolph");
String maxLengthName = null;
int totalWordCount = 0;
for (String name : names) {
    if ( maxLengthName == null ){
        maxLengthName = name;
    }
    if ( name.length() > maxLengthName.length()){
        maxLengthName = name;
    }
    totalWordCount = totalWordCount + name.length();
}
System.out.println("maxLengthName = " + maxLengthName);

如果使用stream來處理，取得字總和也很容易

List<String> names = Arrays.asList("Vincent", "David", "Ken", "Randolph");
long totalWordCount = names.stream().mapToInt(String::length).sum();

但如果想要在同一個stream中取得字數最長的字，試著用跟迴圈一樣的處理方式

String maxLengthName = null;
List<String> names = Arrays.asList("Vincent", "David", "Ken", "Randolph");
long totalLength = names.stream().mapToInt(name ->{
    if ( maxLengthName == null ){
        maxLengthName = name;
    }
    if ( name.length() > maxLengthName.length()){
        maxLengthName = name;
    }
    return name.length();
}).count();

上述的例子中，第4行compile會發生錯誤「Variable used in lambda expression should be final or effectively final」，因為在lambda中改變local variable(maxLengthName)的值是不被允許的，所謂的effective final就是程式中宣告沒有註明變數為final，但變數的值從來沒有被改變過，則可以視為final。

利用AtomicReference來改寫上面的程式，可以避開final的問題，因為這裏maxLengthNameRef並沒有被改變，改變的是裏面存的值。

List<String> names = Arrays.asList("Vincent", "David", "Ken", "Randolph");
AtomicReference<String> maxLengthNameRef = new AtomicReference<>(null);
long totalWordCount = names.stream().peek(name -> {
    maxLengthNameRef.compareAndSet(null, name);
    if (name.length() > maxLengthNameRef.get().length()) {
        maxLengthNameRef.set(name);
    }
}).mapToInt(String::length).sum();
System.out.println("totalWordCount = " + totalWordCount);
System.out.println("maxLength = " + maxLengthNameRef.get());
