Java 基本型別

所謂的型態就是儲存資料的種類，比如Excel中你可以儲存文字、數字、日期不同種類的資料，程式語言中也是如此，目的都是幫你把各種資料放到電腦中進行加減乘除的運算，Java Primitive Type，我們稱為基本型態，一共有８種分別可以使用在整數、小數、文字以及布林值。其中整數的部分有4種可以選擇，小數則有兩種可以選擇。

整數​

整數有以下４種

名稱佔byte數範圍預設值
byte1-272^727 ~ (272^727-1)0-128~127
short2-2152^{15}215 ~ (2152^{15}215-1)0-32768~32767
int4-2312^{31}231 ~ (2312^{31}231-1)0約 -21億~21億
long8-2632^{63}263 ~ (2632^{63}263-1)0-9.2x101810^{18}1018 ~ 9.2x101810^{18}1018 (很大)

為什麼要有這麼多種方式來儲存整數呢？ 因為不同型態的整數佔據記憶體的大小不同，根據你想儲存的資料範圍選擇佔據較小的為原則，例如你想記錄年紀，年紀通常至多百來歲，那就可以選用short，如果你想記錄的是車子的油耗，那short ±\plusmn±3萬多鐵定不夠用，就會採用int。不過實務上通常我們在存整數時，int使用量大概是最高的，畢竟現在記憶體那麼便宜，所以原則上至少挑一個是夠用的。

宣告​

我們會使用「宣告」的語法來要求存放資料到電腦中

語法：

型態　<空格>　變數名稱
int        salary;  
salary = 20000;　//給值
System.out.println("Salary = " + salary);
  • 這裏的型態指的就是long int short byte等等。
  • salary我們稱為「變數名稱」。

宣告語法會在java執行時依據你指定的型態，於記憶體產生相對應大小的空間，以這裏為例我們使用了int，則會產生4 bytes，並且給這個記憶體空間一個名字(salary)以便之後操作。

這裏有幾件事情是需要說明的

  1. 變數給值後才能使用
int salary;
//salary = 20000;　//尚未給值
這例子中我們刻意註解前一行給值的程式，所以你會發現compile時，下面這行程式會報錯
System.out.println("Salary = " + salary); // salary下方會有紅字,因為你用了salary變數，但是沒有先給值
  1. 相同變數名稱不能重覆宣告 
int salary;  
salary = 20000;　//給值
System.out.println("Salary = " + salary);
long salary = 50000;　//前面已經使用過salary，這裏就不能再用一次，否則接下來程式使用salary變數時，會產生語義上的混淆

變數命名規則​

命名規則中包含了一般語法中的制式規定，這部分到是很容易遵守，因為IDE(如eclipse)工具都會幫你做檢查。並不是太大的問題。

  1. 第一個字不能是數字。
  2. 大小寫不同，為不同的變數，例如salary 跟Salary會被視為不同的變數。
  3. 不能使用語法本身的關鍵字。

一般「習慣性」命名規則反而是我們需要去留意的，因為他不是強制性但卻是工作上大家會遵循的一致規則。

  1. 小寫開頭
  2. 如果接第二個字，則後字大寫開頭，例如： int salaryOfMary; O跟M會大寫。
  3. 命名簡短但要有意義

這種命名方式英文叫做「Camel Case」，Camel是駱駝的意思，因為像駝峰一樣會突起，所以稱為Camel Case。

有一些情況變數的命名會有例外，例如「系統常數」的命名通常會採用全大寫，並且中間以 _ (底線)做連接，例如 PAYMENT_TAX，這些「習慣性」的命名規則主要的目的都是希望在閱讀程式時能一眼就看穿這個變數的用途，除了縮短理解程式的時間外同時也可以避免理解錯誤，所以「習慣性」的命名規則是很重要的。

宣告long的數值​

在java中我們寫入一個整數時，compiler預設以int來處理你輸入的數值，因此預設情況下你大概只能輸入 ±\pm±21億之間的整數，如果超過這個範圍，會發生以下的錯誤。

long bigNumber = 2200000000; <- 這裏是２２億，但compile會有錯。
long bigNumber = 2200000000L; 必須加個L在最後，表示用long去處理才不會有問題。

什麼是overflow​

Overflow中文翻譯成叫「資料溢位」，這名詞翻的很道地，就明白的跟你說資料多到滿出來了。

int overflow = 2000000000 + 2000000000;  <- 這裏是20億(2後面9個0）加上20億，正常應該是40億
System.out.println("overflow = " + overflow);
執行結果如下：
overflow = -294967296

結果竟然是一個負值，這其實是因為加起來後其實是４０億，但已經超過了int ±\pm±21億的範圍，導致加相後的位元溢出至正負號位而發生的錯誤。這種錯誤很危險，因為在執行過程中並沒有任何錯誤訊息提示。

想像一下如果你有20億，你再存了20億去銀行，但櫃台跟你說：先生，你現在欠我們銀行2億多。

那宣告成long應該就沒問題了吧

long overflow = 2000000000 + 2000000000;  <- 計算的結果依然是 -294967296

結果依然是負值，原因是運算中有一個規則

若符號兩邊以相同型態的變數運算時(以這個例子來說就是上方的2個20億都是int)，則運算結果也會為同一型態(int)，所以兩個int相加，最後計算的結果還是int，int只能放入21億左右的值，放不下40億，所以計算的結果就還是會有錯。

所以在計算時只能使用long來處理，比如把第一個20億改成long的型態，就是加個L在最後面

long overflow = 2000000000L + 2000000000;  <- 計算的結果 4000000000

當我們把第一個計算的20億改成long的型式，就變成 long + int，這時處理規則就不同了，運算時會把右邊的int也轉成long做計算，就會變成long + long，兩個long相加，結果就會變成是long。就能儲存正確的40億的值了。

Java Language Spec 5.6.2 中有明確定義了在兩個不同型態的變數計算時的運算規則。

浮點數​

就是要儲存小數，有兩種不同的大小可以使用。

型態名稱bytes範圍
float4±3.4x1038 ±1.4x10−45\pm3.4x10^{38}~\pm1.4x10^{-45}±3.4x1038 ±1.4x10−45單精度浮點數，精度約6-7位左右float a = 1.0f;
double8±1.79x10308 ±4.9x10−324\pm1.79x10^{308}~\pm4.9x10^{-324}±1.79x10308 ±4.9x10−324雙精度浮點數，精度約15-16位左右double b = 0.1;

這裏我們來看個例子

float a= 123.4567890123456789f;
double b=123.4567890123456789;
System.out.println(a);//結果 123.45679
System.out.println(b);//結果 123.45678901234568
System.out.println(0.1+0.1+0.1); //0.30000000000000004
  1. 精度上，float最多只能保留7位數字，而double大概15-16位左右，並不是小數點以後而是全部的數字。
  2. 在計算上也會造成誤差，0.1+0.1+0.1，最後卻在小數點很後面的地方多了一個４出來
  3. float 跟double在使用上必須留意，因為有精度上的問題，加減乘除多了會造成精度損失被放大，在計算錢的方面可能不適合，需要小數點精細計算要使用java.math.BigDecimal。
  4. 精度雖然有誤差，但計算速度很快，在能接受誤差的情況下就很適合使用。比如我們的需求只是要記錄一公升汽油可以跑幾公里，有些誤差是無傷大雅的。

文字​

Java中的文字型態是char，每個字２bytes，使用UTF-16編碼，char在使用上指的是單一的一個字，我們通常稱為字元，使用的是單引號。

char oneChar = '中';

單一的字元在使用上當然是很不直覺，使用上我們都是多個字一使用，Java中會以類別 java.lang.String 來處理多個字的組合，使用的符號也會從原本的單引號變成雙引號

String hello = "Hello World";

String雖然是用來儲存多個字，但他並不屬於基本型別的一份子，他是屬於類別的部分。

對Java中的文字採用Unicode有興趣的可以看這裏。

boolean​

所佔用的記憶體大小並沒有明確定義(VM dependent)，只能有兩種值 true false，也就是對或者是錯。

boolean paid = true;
System.out.println("付過錢了沒? "+ paid);
------------------
付過錢了沒? true
